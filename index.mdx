## Overview

Uni Connect is a developer-friendly platform designed for healthcare integration. It helps external clients easily access our APIs, making it simple for them to onboard and authenticate users seamlessly.

---

## How to Get an API Key

To test and access Uni Connect APIs, you need an **API Key**. Here's how to obtain one:

<Steps>
  <Step title="Contact Your Healthcare Provider">
    Reach out to your respective healthcare provider's [Workspace](https://workspace.example.com) to request an API Key. The API Key grants you access to Uni Connect APIs directly without requiring additional client credentials.
  </Step>
  <Step title="Receive Your API Key">
    Once approved, you'll receive an **API Key** — a unique token tied to your account that authenticates all your API requests.
  </Step>
  <Step title="Use the API Key">
    Include the API Key in the header of every API request:
    ```bash
    Authorization: Bearer <your_api_key>
    ```
  </Step>
  <Step title="Need Help?">
    For any issues or detailed information, reach out to us at **info@unimetacare.com**
  </Step>
</Steps>

## How to Get Started with Uni Connect

### 1. Onboarding

**Pre-requisites**

- **API Key**: A unique key assigned to you by your healthcare provider's workspace that authenticates your access to Uni Connect APIs.

**Obtain Your API Key**

- Login to your Uni Care Control Panel at Console and generate your API Key.
- If you are a developer looking to create an app for doctors and patients, you can ask for an API Key from your respective healthcare provider's workspace to get access.
- For any issue or detailed info, reach out to us at **info@unimetacare.com**

---

### 2. Authentication

Uni Connect's API authentication process ensures secure access to sensitive healthcare data using a strong and reliable system. Authentication confirms that the person or application making the request is who they claim to be.

**How to Authenticate with Uni's API**

<Steps>
  <Step title="Use Your API Key">
    Include your API Key in the authorization header of every API request. This tells Uni Connect that your app is authorized to access the data and services.
    ```bash
    Authorization: Bearer <your_api_key>
    ```
  </Step>
  <Step title="Handle Unauthorized Errors">
    If your API Key is invalid or expired, you'll receive a `401 Unauthorized` error. Contact your healthcare provider to renew or reissue your API Key.
  </Step>
</Steps>

## What Comes After Authentication?

Once you've authenticated and successfully integrated with Uni Connect, you can access various features designed to support healthcare integration.

### ABDM Connect

**Introducing the ABDM Connector: A Comprehensive API Suite for Easy Integration.**

The ABDM Connector is designed as a powerful, plug-and-play API suite that simplifies the integration of various healthcare solutions into the Ayushman Bharat Digital Mission (ABDM) ecosystem. Whether you're looking to connect Electronic Health Records (EHR), hospital management systems, or other healthcare platforms, this connector facilitates seamless interoperability.

- **Full Support for All ABDM Functionalities**: Includes creating and linking Health IDs (ABHA), handling consent management, sharing health records, and much more.
- **Flexible Infrastructure Options**: Choose between shared cloud infrastructure for rapid deployment or set up your own private cloud for greater control.

Whether you are a healthcare provider, technology partner, or hospital network, the ABDM Connector ensures a smooth, compliant, and efficient integration experience with the ABDM framework, empowering you to deliver better healthcare services at scale.
